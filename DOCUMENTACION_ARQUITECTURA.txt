STACK TECNOLÓGICO DECISIONAL
================================================================================

BACKEND:
  - NestJS (framework, opinado, DI integrado)
  - TypeScript (type safety, tsconfig strict)
  - PostgreSQL (ACID, JSON, estable)
  - Redis (cache, sessions, queue)
  - TypeORM (database abstraction)
  - Passport.js (JWT auth)
  - class-validator (DTO validation)

FRONTEND:
  - React 18 (UI framework, hooks, concurrent)
  - Vite (build tool, 3-5x más rápido que Webpack)
  - TypeScript (type safety)
  - TailwindCSS (utility-first styling)
  - Redux Toolkit (global state)
  - TanStack Query (server state sync)
  - Axios (HTTP client)

NLP SERVICE:
  - Python 3.11+ (language)
  - FastAPI (web framework)
  - spaCy (NLP library, modelos pre-entrenados)
  - Pandas (data analysis)
  - Celery (async tasks)
  - Redis (queue backend)

MOBILE COMPANION:
  - iOS: Swift + SwiftUI (nativo, SMS capture via Shortcuts)
  - Android: Kotlin + Jetpack Compose (nativo, SMS via BroadcastReceiver)

DEVOPS:
  - Docker (containerization)
  - Kubernetes (orchestration, auto-scaling)
  - Terraform (infrastructure as code)
  - GitHub Actions (CI/CD)
  - Prometheus (metrics)
  - Grafana (visualization)
  - ELK Stack (logging)

================================================================================
PRINCIPIOS ARQUITECTÓNICOS CLAVE
================================================================================

1. CLEAN ARCHITECTURE
   Domain → Application → Infrastructure → Presentation
   ✓ Independencia de framework
   ✓ Testable
   ✓ Escalable a microservicios

2. SOLID PRINCIPLES
   ✓ Single Responsibility
   ✓ Open/Closed
   ✓ Liskov Substitution
   ✓ Interface Segregation
   ✓ Dependency Inversion

3. PRIVACY-FIRST
   ✓ Mínimos permisos necesarios
   ✓ Usuario en control
   ✓ GDPR/COLPRIVADA compliant

4. OFFLINE-FIRST PWA
   ✓ Service Workers para caché
   ✓ Sincronización automática
   ✓ IndexedDB para datos locales

5. HORIZONTAL SCALABILITY
   ✓ Stateless services
   ✓ Database replication
   ✓ Redis cluster
   ✓ Kubernetes auto-scaling

ROADMAP DE 3 FASES
================================================================================

FASE 1: MVP (Actual, 2-3 meses)
  - Monolito modular NestJS
  - React PWA
  - Python NLP service
  - Companion apps (iOS/Android)
  - GitHub Actions CI/CD
  - Target: 3+ bancos, <1000 users

FASE 2: Escala (6-12 meses)
  - Microservicios (Auth, Transaction, Analytics)
  - Kubernetes auto-scaling
  - PostgreSQL replication
  - Redis cluster
  - Custom spaCy NER
  - Target: 10k+ users, 99.9% uptime

FASE 3: Optimización (1-2 años)
  - CQRS + Event Sourcing
  - GraphQL (paralelo a REST)
  - DistilBERT NLP
  - Real-time analytics (BigQuery)
  - Open Banking API
  - Target: 100k+ users
